PaaS 사용자 가이드

개요

PaaS(Platform as a Service)

어플리케이션 런타임 환경과 백엔드 서비스 환경을 제공하는 서비스 입니다.


Java, Go, PHP 등 다양한 언어의 실행환경을 탑재하여 제공하며, MySQL, Redis 등 다양한 백엔드 서비스를 제공합니다.


사용자는 운영체제, 미들웨어에 대한 설치 및 운영에 대한 고민 없이 어플리케이션 배포만으로 안정적인 서비스 운영이 가능합니다.


본 사용자 가이드는 먼저 어플리케이션 데이터 저장을 위한 백엔드 서비스를 생성하고,


빌드팩 기반 어플리케이션과 Docker 기반의 어플리케이션 배포와 서비스 바인딩, 서비스 접근 절차를 소개합니다.


PaaS 서비스 사용 절차



PaaS 서비스를 사용하기 위해서 필요한 절차들이 있습니다.


앱 배포 전 서비스 생성과 일반 앱, Docker 이미지 앱 배포 방법과 앱 배포 후 생성한 서비스 바인딩까지의 절차에 대해서 안내하고 있습니다.


표준 APP.개발 서비스

어플리케이션 데이터를 저장 시키기 위한 백엔드 서비스 생성 방법에 대하여 소개하고 있습니다.


앱 배포(일반 앱), 앱 배포(Docker 앱), 앱관리, 앱 런타임 관리, 서비스 Bind & Unbind에 대하여 소개하고 있습니다.


빌드 팩 기반의 어플리케이션과 Docker 이미지 배포 와 서비스 바인딩에 대하여 소개하고 있습니다.


서비스 생성(서비스 메뉴)


사용자가 개발한 어플리케이션에는 데이터베이스 및 세션을 사용하는 환경의 어플리케이션이 있습니다.


PaaS 에서는 그런 데이터베이스 및 세션을 지원해 주기위해서 서비스를 제공하고 있습니다.


사용자는 어플리케이션에 알맞은 설정을 통해 PaaS 에서 지원해 주는 서비스와 연결 후 어플리케이션에서 필요한 데이터베이스로 사용할 수 있습니다.


이번 순서에서는 서비스 생성 절차에 대해서 안내하고 있습니다.


(어플리케이션에서 서비스 연결 설정에 관한 가이드는 개발 가이드를 참고 하시기 바랍니다.)


서비스 만들기 페이지
  • 서비스 대시보드 하단에 위치한 [+]버튼을 클릭합니다.
  • 우측에 [서비스 만들기]페이지가 나타납니다.
서비스 정보 입력
  • 서비스 추가 팝업에 정보들을 순서에 맞게 알맞게 입력합니다.
  • 서비스 선택, 서비스 이름 입력, 서비스 플랜 선택 순서로 진행합니다.
  • 서비스 정보 입력이 완료 되면 [생성]버튼을 클릭하여 서비스를 생성합니다.
서비스 생성 완료
  • 서비스가 성공적으로 생성된 것을 확인 할 수 있습니다.
앱 배포(일반 앱 배포)
앱 생성 페이지 호출 - 이미지형
  • [서비스 대시보드]를 클릭해서 대시보드로 이동합니다.
  • 서비스 대시보드 중앙에 위치한 [+]버튼을 클릭하면 우측에 [APP 개발 서비스 추가]페이지가 나타납니다.
앱 생성 페이지 호출 - 리스트형
  • [서비스 대시보드]를 클릭해서 대시보드로 이동합니다.
  • [리스트] 버튼을 클릭합니다.
  • 서비스 대시보드 우측에 위치한 [+]버튼을 클릭하면 우측에 [APP 개발 서비스 추가]페이지가 나타납니다.
앱 생성 정보 입력
  • APP 개발 서비스 추가 팝업에 정보들을 순서에 맞게 알맞게 입력합니다.
  • 앱 유형 선택(일반앱), 개발언어 선택, 앱 명 입력, 호스트 URL입력, APP Package 파일 업로드합니다.
  • 앱 정보 입력이 완료 되면 [다음]버튼을 클릭하여 세부 설정 페이지로 이동합니다.
앱 생성 세부설정
  • [세부설정하기]버튼을 클릭하면 세부설정 가능한 슬라이드가 나타납니다.
  • 서비스 세부설정이 완료되면 [배포]버튼을 클릭하여 앱을 배포합니다.
앱 생성 완료
  • 앱 배포가 성공적으로 완료되면 앱 상세 페이지로 이동하게 됩니다.
  • APP 런타임 페이지에서 태가 STARTED 상태로 변하게 되면 앱이 실행된 것입니다.
  • APP 런타임 페이지 중앙에 위치한 URL을 클릭하면 앱이 생성된 것을 확인할 수 있습니다.

앱 런타임 관리

앱 런타임 페이지 이동
  • 서비스 대시보드에서 확인하고자 하는 앱 판넬을 클릭하여 APP 런타임 페이지로 이동합니다.
앱 주요 정보

  • 생성된 APP 의 주요 정보 확인합니다.
  • 해당 APP 의 CPU, Disk, Memory 사용량 표시합니다.
  • 해당 APP 제어합니다.
  • [시작/중지]버튼을 클릭하면 앱을 시작하고 중지한다.
  • [새로고침]버튼을 클릭하면 페이지를 새로고침합니다.
  • [재시작]버튼을 클릭하면 앱이 재시작됩니다.
  • [삭제]버튼을 클릭하면 앱이 삭제됩니다.
  • [이름변경]버튼을 클릭하면 앱의 이름을 변경 할 수 있습니다.
인스턴스 재배포
  • 앱 상세 정보 하단에 위치한 [재배포]버튼을 클릭합니다.
  • 우측에 [인스턴스 재배포]페이지가 나타납니다.
  • [리소스 변경하기]버튼을 클릭하면 리소스를 변경할 수 있는 창이 나타납니다.
  • 슬라이드 바를 움직여 리소스를 변경합니다.
  • 슬라이드 바를 움직이는 대로 배포할 리소스 정보가 변경됩니다.
  • 리소스를 조절한 후 [재배포]버튼을 클릭합니다.
  • 앱 런타임에서 변경된 리소스를 확인합니다.
인스턴스 상태 확인(이미지형)
  • APP 런타임으로 이동하면 인스턴스 상태는 기본적으로 이미지형으로 보여줍니다.
  • "STARTED"는 인스턴스가 정상 작동중인 상태입니다.
  • "STARTING"는 인스턴스가 시작되고 있는 상태입니다.
  • "FAILED"는 인스턴스가 오류 난 상태입니다.
  • 인스턴스 상태를 확인할 수 있습니다.
  • 해당 APP 의 인스턴스 총 수를 확인할 수 있습니다.
  • 인스턴스의 고유 번호를 확인할 수 있습니다.
  • 인스턴스의 CPU, 디스크, 메모리 사용량을 확인할 수 있습니다.
  • 인스턴스 재시작 할 수 있습니다.
인스턴스 상태 확인(리스트형)
  • [리스트형]버튼을 클릭하면 리스트형 인스턴스 상태를 보여줍니다.
모니터링
  • 해당 앱의 CPU, Memory, Disk 모니터링 상태를 보여줍니다.
  • [확대]버튼을 클릭하면 모니터링 탭이 확대됨을 확인할 수 있습니다.
  • [축소]버튼을 클릭하면 모니터링 탭이 축소됨을 확인할 수 있습니다.
라우트
  • 라우트 탭에서 [+]버튼을 클릭합니다.
  • 라우트 추가 팝업이 뜨고 호스트 명에 값을 입력합니다.
  • [확인]버튼을 클릭하면 추가가 되며 로딩바가 표시됩니다.
  • 새로고침이 끝난 후 추가된 라우트를 확인합니다.
  • [확대]버튼을 클릭하면 라우트 탭이 확대됨을 확인할 수 있습니다.
  • [축소]버튼을 클릭하면 라우트 탭이 축소됨을 확인할 수 있습니다.
  • [휴지통]버튼을 클릭하면 삭제할 것인지 확인하는 창이 나타납니다.
  • [예]버튼을 클릭하면 라우트가 삭제됩니다.
이벤트 로그
  • 해당 앱에서 어떤 이벤트가 일어났는지 확인할 수 있습니다.
  • [확대]버튼을 클릭하면 이벤트 로그가 확대됨을 확인할 수 있습니다.
  • [축소]버튼을 클릭하면 이벤트 로그가 축소됨을 확인할 수 있습니다.
환경변수
  • 환경변수 탭에서 [+]버튼을 클릭합니다.
  • 환경변수 추가 팝업이 뜨고 변수 이름과 변수 값에 적절한 값을 입력합니다.
  • [확인]버튼을 클릭하면 추가가 되며 로딩바가 표시됩니다.
  • 새로고침이 끝난 후 추가된 환경변수를 확인합니다.
  • [확대]버튼을 클릭하면 환경변수 탭이 확대됨을 확인할 수 있습니다.
  • [축소]버튼을 클릭하면 환경변수 탭이 축소됨을 확인할 수 있습니다.
  • [연필]버튼을 클릭하면 수정할 수 있는 팝업 창이 나타납니다.
  • 변수 값에 수정한 값을 입력 후 [확인] 버튼을 클릭하면 환경변수가 수정됩니다.
  • [휴지통]버튼을 클릭하면 삭제할 것인지 확인하는 창이 나타납니다.
  • [예]버튼을 클릭하면 환경변수가 삭제됩니다.
SSH
  • 해당 앱의 인스턴스 별 SSH 접속하여 관련 내용을 확인할 수 있습니다.
  • 기본 값은 "선택 없음"으로 되어 있습니다.
  • [확대]버튼을 클릭하면 SSH 탭이 확대됨을 확인할 수 있습니다.
  • [축소]버튼을 클릭하면 SSH 탭이 축소됨을 확인할 수 있습니다.

서비스 Bind & Unbind

연결 서비스 관리 리스트
  • 서비스 대시보드에서 생성한 서비스의 총 개수 출력합니다.
  • 서비스 대시보드에서 생성한 서비스들의 이름과 이미지 출력합니다.
서비스 연결
  • 서비스 대시보드에서 생성한 서비스가 해당 앱과 연결이 안 되어있을 경우 이미지가 희미하게 보입니다.
  • [· · ·]버튼을 클릭하면 선택 옵션이 나타납니다.
  • 서비스 대시보드에서 생성된 해당 서비스의 정보를 보여줍니다.
  • 서비스 대시보드에서 생성한 서비스가 해당 APP 과 연결이 안 되어있을 경우 이미지가 희미하게 보입니다.
  • [연결하기]버튼을 클릭하면 팝업 창이 나타나며 서비스를 바인딩할 것인지 묻습니다.
  • [예]버튼을 클릭하면 서비스가 해당 앱과 바인딩 됩니다.
  • 앞서 바인딩한 서비스를 보면 해당 서비스의 이미지가 선명하게 보이는 것을 확인할 수 있습니다.
  • [서비스 정보보기]버튼을 클릭하면 팝업 창이 나타납니다.
  • 서비스와 앱 간 연결된 정보가 보입니다.
서비스 연결 해제
  • [연결하기]버튼을 클릭하면 바인딩을 해제할 지 확인하는 창이 나타납니다.
  • [예]버튼을 클릭하면 바인딩이 해제되며 이미지가 다시 희미해진 것을 확인할 수 있습니다.

앱 로깅

  • 앱 별 로그를 확인할 수 있습니다.
