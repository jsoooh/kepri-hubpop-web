표준APP.개발 활용하기

개요

표준APP.개발은 어플리케이션 런타임 환경과 백엔드 서비스 환경을 제공하는 PaaS(Platform as a Service)서비스 입니다.

Java, Go, PHP 등 다양한 언어의 실행환경을 탑재하여 제공하며, MySQL, Redis 등 다양한 백엔드 서비스를 제공합니다. 사용자는 운영체제, 미들웨어에 대한 설치 및 운영에 대한 고민 없이 어플리케이션 배포만으로 안정적인 서비스 운영이 가능합니다.

본 사용자 가이드는 먼저 어플리케이션 데이터 저장을 위한 백엔드 서비스를 생성하고, 빌드팩 기반 어플리케이션과 Docker 기반의 어플리케이션 배포와 서비스 바인딩, 서비스 접근 절차를 소개합니다.

서비스 생성
어플리케이션 데이터를 저장 시키기 위한 백엔드 서비스 생성 방법에 대하여 2 가지 방법으로 나누어 소개하고 있습니다.
앱 배포(일반 앱), 앱 배포(docker 앱), 앱관리, 앱 런타임 관리, 서비스 Bind & Unbind
빌드 팩 기반의 어플리케이션과 Docker 이미지 배포 와 서비스 바인딩에 대하여 소개하고 있습니다.



PaaS 서비스를 사용하기 위해서 필요한 절차들이 있습니다.
앱 배포 전 서비스 생성과 일반 앱, docker 이미지 앱, CLI를 통한 앱 배포 방법과
앱 배포 후 생성한 서비스 바인딩까지의 절차에 대해서 안내하고 있습니다.

서비스란?
사용자가 개발한 어플리케이션에는 데이터베이스 및 세션을 사용하는 환경의 어플리케이션이 있습니다.
PaaS 에서는 그런 데이터베이스 및 세션을 지원해 주기위해서 서비스를 제공하고 있습니다.
사용자는 어플리케이션에 알맞은 설정을 통해 PaaS 에서 지원해 주는 서비스와 연결 후
어플리케이션에서 필요한 데이터베이스로 사용할 수 있습니다.
이번 순서에서는 서비스 생성 절차에 대해서 안내하고 있습니다.

(어플리케이션에서 서비스 연결 설정에 관한 가이드는 개발 가이드를 참고 하시기 바랍니다.)

서비스 생성(서비스 메뉴)

프로젝트 선택


1) 프로젝트 선택
① Hub-PoP 포탈에 로그인을 합니다.
② Hub-PoP 프로젝트를 클릭합니다.

표준 APP 개발 이동


2) 표준 APP 개발 -> 대쉬보드 페이지 이동
① Hub-PoP 로그인 후 프로젝트를 선택한 후 좌측메뉴에서 [표준 APP 개발]메뉴의 [대쉬보드]를 클릭합니다.

앱 배포(일반 앱 배포)
APP 생성 페이지 호출 - 이미지형


1) APP 생성 페이지 호출
① 표준 APP 개발 대시보드 중앙에 위치한 [+] 버튼을 클릭하면 우측에 [APP 개발 서비스 추가]페이지가 나타납니다.

APP 생성 정보 입력


2) APP 생성 정보 입력
① APP 개발 서비스 추가 팝업에 정보들을 순서에 맞게 알맞게 입력합니다.
    앱 유형 선택(일반앱), 개발언어 선택, 앱 명 입력, 호스트 URL입력, APP Package 파일 업로드, WAS 선택
    * APP Package 파일 업로드 시 파일 첨부는 필수가 아님.
② 서비스 정보 입력이 완료 되면 [다음] 버튼을 클릭하여 세부 설정 페이지로 이동합니다.

APP 생성 세부설정


3) APP 생성 세부설정(리소스 정보 설정)
① [세부설정하기] 버튼을 클릭하면 세부설정 가능한 슬라이드가 나타납니다.
② 서비스 세부설정이 완료 되면 [배포] 버튼을 클릭하여 APP을 배포합니다.

APP 생성 완료


4) APP 생성완료
① APP배포가 성공적으로 완료되면 앱 상세 페이지로 이동하게 됩니다.
② APP 런타임 페이지에서 시작 상태로 변하게 되면 APP이 실행된 것입니다.

서비스 만들기 페이지


1) 연결 서비스 만들기 페이지 호출
① 연결 서비스 대시보드 하단에 위치한 [+] 버튼을 클릭하면 우측에 [서비스 만들기]페이지가 나타납니다.

서비스 정보 입력


2) 서비스 생성
① 서비스 추가 팝업에 정보들을 순서에 맞게 알맞게 입력합니다.
    서비스 선택, 서비스 이름 입력, 서비스 플랜 선택 순서로 입력
② 서비스 정보입력이 완료 되면 [생성] 버튼을 클릭하여 서비스를 생성 합니다.

서비스 생성 완료


3) 서비스 생성 완료
① 서비스가 생성되고 나면 연결 서비스 항목과 같이 서비스가 성공적으로 생성된 것을 확인 할 수 있습니다.

APP과 DB 인스턴스 연결


1) 연결 서비스 메뉴 호출
① 대시보드 하단에 위치한 DB정보의 ... 버튼을 클릭하면 확장메뉴에 [연결하기]메뉴가 나타납니다.



2) 서비스 연결 확인
① 서비스를 연결하시겠습니까? 확인창이 호출되며, 확인 클릭시 APP을 재시작하게 됩니다.

APP과 DB 인스턴스 연결 완료


3) 서비스 생성 완료
① 서비스가 생성되고 나면 서비스가 성공적으로 생성된 것을 확인 할 수 있습니다.

앱 런타임 관리

앱 런타임 페이지 이동


1) APP 런타임 페이지 이동


① 서비스 대시보드에서 확인하고자 하는 앱 판넬을 클릭하여 APP 런타임 페이지로 이동합니다.

구조
    ①
  • 생성된 APP 의 주요 정보와 해당 APP의 CPU, Disk, Memory 사용량 표시
  • 해당 APP 의 인스턴스 재배포 기능
    ②
  • 이미지형, 리스트형으로 해당 APP 의 인스턴스 개수 및 상태(시작, 시작중, 실패) 를 표시
    ③
  • 해당 APP 의 라우트 확인 및 추가, 변경, 삭제 기능
  • 해당 APP 의 이벤트 로그 확인
  • 해당 APP 의 환경변수 확인 및 추가, 변경, 삭제 기능
  • 해당 APP 의 인스턴스 별로 SSH 확인

    ④
  • 생성된 서비스를 해당 APP 과 바인딩 및 언바인드 하는 기능
  • 서비스 정보 및 바인딩 정보 확인 기능
앱 주요 정보 및 인스턴스 재배포

1) APP 주요 정보


① 생성된 APP 의 주요 정보 확인
② 해당 APP 의 CPU, Disk, Memory 사용량 표시
③ 해당 APP 제어

  • 시작/중지 버튼 : 앱을 시작하고 중지한다.
  • 새로고침 버튼 : 앱을 새로고침 한다.



  • 실행 설정 버튼


    • - 재시작 : 앱을 restart 한다.
    - 삭제 : 앱을 삭제한다.
    - 이름변경 : 앱의 이름을 변경한다.



2) 인스턴스 재배포 Layer 창 호출




① App 상세 정보 하단에 위치한 [ 재배포 ] 버튼을 클릭하면 우측에 [인스턴스 재배포] Layer 창이 나타납니다.


② App 상세 정보 하단에 위치한 [리소스 변경하기] 버튼을 클릭하면 리소스를 변경할 수 있는 창이 나타납니다. 슬라이드 바를 움직여 리소스를 변경합니다.
③ 슬라이드 바를 움직이는 대로 재배포할 리소스 정보가 변경됩니다.
④ 리소스를 조절한 후 [재배포] 버튼을 클릭합니다.
⑤ APP 런타임에서 변경된 리소스를 확인합니다.


인스턴스 상태 확인(이미지형)


1) 인스턴스 상태 확인(이미지형)


① APP 런타임으로 이동하면 기본적으로 인스턴스 상태는 이미지형을 보여줍니다.
② 정의된 인스턴스 상태입니다.

- 시작 : 인스턴스가 정상작동 중인 상태
- 시작중 : 인스턴스가 시작되고 있는 상태
- 실패 : 인스턴스가 오류 난 상태

③ 인스턴스 상태를 확인할 수 있습니다.


- ①해당 APP 의 인스턴스 총 개수
- ②인스턴스의 고유 번호
- ③인스턴스의 CPU, 디스크, 메모리 사용량
- ④인스턴스 재시작


인스턴스 상태 확인(리스트형)


1) 인스턴스 상태 확인(리스트형)


① 리스트형 탭을 클릭하면 리스트형 인스턴스 상태를 보여줍니다.


서비스 Bind & Unbind

연결 서비스 관리 리스트


① 서비스 대시보드에서 생성한 서비스의 총 개수 출력 및 서비스 대시보드에서 생성한 서비스들의 이름과 이미지 출력.



서비스 연결


① 실행 설정 영역에서 연결하기버튼을 클릭하면 연결 팝업 창 이 나타나며 서비스를 바인딩할 것인지 묻습니다.
③ 확인버튼을 클릭하면 서비스가 해당 APP 과 바인딩 됩니다.





① 앞서 바인딩한 서비스를 보면 해당 서비스의 이미지가 선명하게 보이는 것을 확인할 수 있습니다.
② 서비스 정보보기버튼을 클릭해서 서비스 팝업 정보 창을 보면 연결 앱 수가 1로 증가한 것을 확인할 수 있습니다.
③ 연결정보버튼을 클릭하면 연결 정보 팝업 창이 나타납니다. 서비스와 APP 간 연결된 정보가 보입니다.



서비스 연결 해제


① 연결 해제버튼을 클릭하면 바인딩을 해제할 지 확인하는 창이 나타납니다.
② 서비스 해제 팝업 창이 나타나며 [ 예 ] 버튼을 클릭하면 바인딩을 해제 할 수 있습니다..



환경변수


1) 환경변수 추가


① ①[ + ] 버튼을 클릭합니다.
② 환경변수 추가 팝업이 뜨고 ②변수 이름과 ③변수 값에 적절한 값을 입력합니다.
③ ④[ 확인 ] 버튼을 클릭하면 추가가 되며 로딩바가 표시됩니다.
④ 새로고침이 끝난 후 추가된 환경변수를 확인합니다.





2) 환경변수 확대 및 축소


① ①[ 확대 ] 버튼을 클릭하면 라우트가 확대됨을 확인할 수 있습니다.
② ②[ 축소 ] 버튼을 클릭하면 라우트가 축소됨을 확인할 수 있습니다.





3) 환경변수 수정


① ①[ 연필 ] 버튼을 클릭하면 수정할 수 있는 팝업 창이 나타납니다.
② ②변수 값에 수정한 값을 입력 후 ③[ 확인 ] 버튼을 클릭하면 환경변수가 수정됩니다.




4) 환경변수 삭제


① ①[ 휴지통 ] 버튼을 클릭하면 삭제할 것인지 확인하는 창이 나타납니다.
② ②[ 예 ] 버튼을 클릭하면 환경변수가 삭제됩니다.


SSH


1) SSH


① 해당 앱의 인스턴스 별 SSH 접속하여 관련 내용을 확인할 수 있습니다. 기본 값은‘선택 없음' 으로 되어 있습니다.





2) SSH 확대 및 축소


① ①[ 확대 ] 버튼을 클릭하면 SSH 가 확대됨을 확인할 수 있습니다.
② ②[ 축소 ] 버튼을 클릭하면 SSH 가 축소됨을 확인할 수 있습니다.



라우트 추가


1) 라우트 추가


① [ + ] 버튼을 클릭합니다.
② 라우트 추가 팝업이 뜨고 호스트 명 값을 입력합니다.
③ [ 확인 ] 버튼을 클릭하면 추가가 되며 로딩바가 표시됩니다.
④ 새로고침이 끝난 후 추가된 라우트를 확인합니다.





2) 라우트 확대 및 축소


① [ 확대 ] 버튼을 클릭하면 라우트가 확대됨을 확인할 수 있습니다.
② [ 축소 ] 버튼을 클릭하면 라우트가 축소됨을 확인할 수 있습니다.





3) 라우트 삭제


① [ 휴지통 ] 버튼을 클릭하면 삭제할 것인지 확인하는 창이 나타납니다.
② [ 예 ] 버튼을 클릭하면 라우트가 삭제됩니다.


이벤트 로그


1) 이벤트 로그


① 해당 앱에서 어떤 이벤트가 일어났는지 로그로 확인할 수 있습니다. ② ①[ 확대 ] 버튼을 클릭하면 이벤트 로그가 확대됨을 확인할 수 있습니다.
③ ②[ 축소 ] 버튼을 클릭하면 이벤트 로그가 축소됨을 확인할 수 있습니다.


앱 로깅



① APP 런타임 화면에서 ①APP 로깅을 클릭하면 앱 별 로그를 확인할 수 있습니다.
