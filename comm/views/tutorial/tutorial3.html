CloudXpert를 보다 편리하게 사용하기 위해 회원가입 및 로그인 방법, 전반적인 서비스 개요를 파악하시면 더욱 유용하게 즐길 수 있습니다.

회원가입과 로그인

CloudXpert 서비스를 사용하기 위해서는 회원이 되는 절차와 회원 가입 후 로그인을 하는 절차가 필요로 합니다.
CloudXpert 서비스의 회원 유형은 시스템 관리자, 프로젝트 책임자, 사용자(작업 관리자, 작업 사용자)로 분류됩니다.

회원가입

사용자의 경우 회원가입을 통해 사용자 등록을 완료합니다.

  • ① 첫 화면인 로그인 화면에서 [Membership Join] 을 누릅니다.
  • ② 회원가입 정보를 입력합니다.
  • ③ [Sign Up] 을 클릭하여 가입을 완료합니다.
로그인

로그인을 통해 서비스를 시작할 수 있습니다.

  • ① 아이디, 비밀번호를 입력합니다.
  • ② [Sign In] 을 클릭하여 로그인합니다.

대시보드

로그인을 통한 후 처음으로 나타나는 화면으로 회원 유형(프로젝트 책임자, 사용자)에 따라 보여지는 모습이 다릅니다.

프로젝트 책임자 대시보드

공통 정보 외에 프로젝트 정보를 보여 줍니다.


       
  • ① 로그인 사용자가 권한을 가진 프로젝트의 목록과 정보를 보여줍니다.
    클릭 시 프로젝트 상세 화면으로 전환됩니다.
  • ② 클릭 시 좌측의 메뉴 목록이 나타나거나 사라진다.
  • ③ 로그인 사용자에게 전달되는 Alarm 목록이 나타나고 더보기 [>] 클릭 시 모니터링의 알람 목록 화면으로 전환됩니다.
         알람 목록 : 로그인 사용자에게 전달되는 Alarm 목록이 나타나고 더보기 [>] 클릭 시 모니터링의 알람 목록 화면으로 전환됩니다.
         Document View : 화면 가이드 화면으로 전환됩니다.
         Notice Info : 공지사항 화면으로 전환됩니다.
  • ④ 화면의 상단에 보여지는 메뉴입니다.
         알람 목록 : 알람 목록이 나타나고 [알람 더보기] 클릭 시 모니터링의 알람 목록 화면으로 전환됩니다.
         언어설정 : 한국어, 영어 클릭 시로 해당 언어로 전환된 사이트를 볼 수 있습니다.
         튜토리얼 : 화면 가이드 화면으로 전환됩니다.
         사용자 및 공지사항 로그아웃 : 개인정보 수정 및 회원 탈퇴, 공지사항 목록, 로그아웃 등의 화면, 기능을 나타냅니다.
  • ⑤ 아이콘 클릭 시 슬라이드로 나타나며 항목 클릭과 해제로 대시보드 화면을 설정합니다.
         알림/문서보기/공지사항 클릭과 해제로 ③ 항목을 보이거나 보이지 않도록 설정합니다.
  • ⑥ 프로젝트 정보란에 나타나는 프로젝트 별로 배경색을 설정합니다. 화면 전환 후나 다음 로그인 시에 변경된 정보로 보여집니다.
사용자 대시보드

공통 정보 외에 작업 정보를 보여 줍니다.

  • ① 로그인 사용자가 권한을 가진 작업의 목록과 정보를 보여줍니다.
    클릭 시 작업 상세 화면으로 전환됩니다.

프로젝트 관리

플랫폼 서비스를 이용하기 위한 작업을 생성하기 위해 먼저 프로젝트를 등록 및 관리합니다.
프로젝트 하위에는 다건의 작업과 멤버를 생성하여 관리할 수 있습니다.

프로젝트 목록

로그인 사용자가 권한을 가진 프로젝트 목록을 볼 수 있습니다.
[프로젝트 등록]은 프로젝트 책임자 권한을 가진 사용자만 사용할 수 있습니다.

  • 프로젝트의 간략 정보를 볼 수 있습니다.
  • 하위 작업 건수와 멤버 수를 볼 수 있습니다.
  • ① 프로젝트 책임자 권한을 가진 사용자만 [프로젝트 등록]이 활성화 됩니다.
프로젝트 상세

프로젝트의 상세내용을 볼 수 있습니다.
일반정보와 프로젝트이력/용량변경 요청 내역, 하위 작업 목록, 멤버 목록을 보여주고 관리할 수 있습니다.
하위 작업 중 생성 에러가 발생된 작업의 경우 ①과 같은 붉은색 테두리로 표현됩니다.

프로젝트 멤버 등록 및 삭제

현재 프로젝트의 멤버를 관리합니다. 하위 작업의 멤버도 프로젝트의 멤버에 등록된 목록 내에서 선택 가능합니다.
프로젝트 책임자 권한을 가진 사용자가 등록, 삭제가 가능합니다.

  • 프로젝트 책임자 권한을 가진 사용자만 멤버를 등록하는 버튼(①), 삭제하는 아이콘(②)이 활성화 됩니다.
  • 전체 사용자 중 현재 프로젝트 멤버 외의 사용자의 목록이 보여지고 체크 클릭 후 [확인] 클릭으로 등록 가능합니다.
  • ① 역할 란을 관리자로 설정 시 작업 관리자로 권한이 부여되고 사용자로 설정 시 작업 사용자로 권한이 부여됩니다.
  • 작업 관리자로 권한이 부여되면 본인이 할당된 작업의 관리가 가능해 집니다.
용량변경 요청

프로젝트가 생성될 때 기본으로 부여된 프로젝트 용량이 할당됩니다.
하위 작업은 프로젝트 용량의 범위 내에서만 생성이 가능합니다.
하위 작업은 생성 시 쿼터가 부족할 경우 프로젝트 용량 변경 요청을 할 수 있습니다.

  • 프로젝트 책임자 권한을 가진 사용자만 ① 변경 요청 버튼이 활성화 됩니다.
  • 프로젝트 책임자에 의한 프로젝트 요청은 시스템 관리자가 승인 가능합니다.
  • 변경 요청할 용량 선택, 사유를 입력 후 요청 등록합니다.
작업 목록 및 등록

하위 작업을 조회하고 관리할 수 있습니다.

  • 프로젝트의 하위 작업 목록을 보여줍니다.
  • 작업이 정상적으로 생성된 경우 상단이 ① 초록색으로 보이고 정상 생성되지 않은 경우 ② 붉은 테두리를 볼 수 있습니다.
  • 작업이 정상 생성되지 않은 경우는 플랫폼 서비스의 표준 APP.개발 서비스나 서버 가상화 서비스 시스템으로 연계 정보를 보낼 때 제대로 생성안된 경우입니다.
  • 프로젝트 책임자 권한을 가진 사용자만 등록 가능합니다.
  • ① 작업 쿼터는 프로젝트 쿼터와 하위 작업 쿼터들의 합을 계산하여 등록 가능한 쿼터만 선택됩니다.
프로젝트 수정 및 삭제

해당 프로젝트를 수정 또는 삭제할 수 있습니다.
삭제 시에는 하위에 작업이 존재할 경우는 작업 삭제 후 진행 가능합니다.

  • 프로젝트 책임자 권한을 가진 사용자만 ② 수정, ① 삭제 가능합니다.
  • ① 삭제 시에는 하위에 작업이 존재할 경우는 작업 삭제 후 진행 가능합니다.

작업 관리

작업은 플랫폼 서비스의 표준 APP.개발 서비스나 서버 가상화 서비스를 이용하기 위한 기본 단위입니다.
작업을 생성할 때 표준 APP.개발 서비스나 서버 가상화 서비스에서도 인스턴스 생성 등이 가능해 집니다.

작업 목록 및 등록

로그인 사용자에 할당된 작업 목록이 보여지고 프로젝트 책임자나 작업 관리자 권한의 사용자가 관리 가능합니다.
정상적으로 생성된 작업의 경우는 ① 테두리가 초록색으로 생성 오류의 경우는 ② 붉은색으로 표현됩니다.

  • 작업의 간략정보가 기록된 목록을 볼 수 있습니다.
  • 정상적으로 생성된 작업의 경우는 테두리가 초록색으로 생성 오류의 경우는 붉은색으로 표현됩니다.
  • 프로젝트 책임자 권한의 사용자만 ① [작업 등록] 이 활성화 됩니다.
  • 우측 팝업되는 작업 등록 창의 ② 작업 쿼터는 프로젝트 쿼터와 프로젝트 하위의 작업 쿼터의 합을 계산한 후 신청 가능한 쿼터만 선택 가능합니다.
작업 상세

작업의 상세 정보를 볼 수 있는 화면으로 기본정보, 쿼터, 멤버, 작업이력/용량변경 요청 및 표준 APP.개발 서비스나 서버 가상화 서비스 에서 생성된 인스턴스 목록을 볼 수 있습니다.

  • 작업의 쿼터 정보는 전체 신청 가능한 최고 용량 외 현재 신청된 용량을 보여줍니다.
  • 작업 생성 시 ① 최초 생성이라는 항목으로 한건이 초기 생성됩니다.
  • 초기생성이나 용량변경 요청을 하고 승인되면 플랫폼 서비스로 다시 연계되어 변경될 때 연계 실패 시 ② 연계실패로 보여집니다.
  • ③ [재생성] 클릭 시 플랫폼 서비스로 다시 연계처리 됩니다.
  • 해당 작업으로 표준 APP.개발 서비스의 서비스가 추가된 경우 목록이 보여지고 클릭 시 상세내역을 볼 수 있습니다.
  • 해당 작업으로 서버 가상화 서비스의 서비스가 추가된 경우 목록이 보여지고 클릭 시 상세내역을 볼 수 있습니다.
작업 멤버 등록 및 삭제

작업 멤버를 등록 관리합니다.
작업 멤버 등록을 통해 표준 APP.개발 서비스나 서버 가상화 서비스를 사용할 수 있는 권한이 부여됩니다.

  • 작업 멤버의 등록과 삭제는 프로젝트 책임자나 작업 관리자로 권한 부여된 사용자만 가능합니다.
작업 용량변경 요청 등록 및 요청 승인

표준 APP.개발 서비스나 서버 가상화 서비스를 사용하며 추가 용량이 필요한 경우 요청할 수 있습니다.
작업 상세의 쿼터 정보를 보면 필요 용량을 가늠해 볼 수 있습니다.

  • 작업 용량변경 요청은 작업과 관련된 사용자는 모두 신청 가능하고 승인은 프로젝트 책임자와 작업 관리자 권한의 사용자가 처리합니다.
  • 용량변경 요청의 ① 용량은 프로젝트 쿼터와 전체 작업 쿼터를 계산하여 가능한 용량만 신청 가능합니다.
작업 수정 및 삭제

작업 기본정보를 수정하거나 작업을 삭제합니다.
표준 APP.개발 서비스나 서버 가상화 서비스에서 사용하고 있는 인스턴스나 서비스가 있는 경우는 해당 서비스를 삭제 후 삭제 가능합니다.
