개요

모니터링(Monitoring)

인스턴스를 생성하거나 어플리케이션을 배포했을 때 문제가 발생했는지에 대한 여부를 데이터 저장 및 화면에 출력하는 서비스 입니다.


사용자는 작업을 선택, 인스턴스를 생성하고 통합 모니터링에서는 인스턴스 사용량 데이터를 화면에 출력합니다.


어플리케이션의 배포 상태와 현시점 사용량 현황, 백엔드 서비스 상태를 확인할 수 있습니다.


작업에 포함된 인스턴스와 어플리케이션 상태에 대한 이벤트와 로그를 확인할 수 있습니다.


사용자 별로 해당 작업 인스턴스와 어플리케이션 상태에 대한 사용치 알람을 확인하여 조치할 수 있습니다.


대시보드 확인하기

프로젝트 하위 작업에 대한 VM 사용량 및 서버 가상화와 표준 APP 개발 서비스 WEB, WAS, DB의 서비스 상태와 알람 및 이벤트에 대한 내역을 확인할 수 있습니다.

  • 현재 선택된 프로젝트 및 작업을 변경하여 다른 프로젝트 및 작업의 데이터를 확인할 수 있습니다.
  • IaaS 가상머신이나 PaaS 앱의 Start, Stop, Error 상태를 보여줍니다.
  • IaaS 가상머신의 CPU, MEMORY, DISK 사용량과 DISK IOPS와 NETWORK IO 추이를 보여줍니다.
  • 가상머신 CPU 사용량 기준으로 가장 많이 사용한 VM TOP 5를 상단에서 확인할 수 있습니다.
  • 앱 CPU 사용량 기준으로 가장 많이 사용한 앱 TOP 5를 상단에서 확인할 수 있습니다.
  • 알람과 이벤트는 하루를 기준으로 대시보드에 보여지고 있습니다.
구분 CPU MEMORY DISK DISK IOPS NETWORK IO
정의
  • 전체 VM 및 APP의 평균 CPU 사용량 통계
  • 전체 VM 및 APP의 평균 MEMORY 사용량 통계
  • 전체 VM 및 APP의 평균 DISK 사용량 통계
  • 디스크로부터 Input/Out을 수행한 수치를 의미(초당 입출력 횟수)
  • 네트워크의 입출력 데이터(Megabyte기준)
가상서버 Top 뷰

현재 작업의 CPU 사용량을 기준으로 가상서버를 나열한 화면입니다. 각 가상머신 이름, IP, CPU, MEMORY, DISK 사용량과 DISK IOPS, NETWORK IO를 확인할 수 있습니다.

가상머신 모니터링

각 가상머신에 대한 CPU, MEMORY, DISK, NETWORK 사용량을 확인할 수 있습니다.

  • 좌측 리스트는 인스턴스의 종류와 VM명을 포함하고 있습니다.(현재 선택된 상단화면은 인스턴스 유형이 Load Balance인 VM이름 abc_01이다.)
  • 모니터링 내역으로는 CPU(Max, Avg, Min의 사용량), MEMORY(Max, Avg, Min의 사용량), Disk 사용량, NETWORK 송수신 바이트와 패킷으로 구성되어있습니다.

App Top 뷰

현재 작업의 CPU 사용량을 기준으로 가상서버를 나열한 화면입니다. 각 앱의 이름, IP, CPU, MEMORY, Disk 사용량과 DISK IOPS, NETWORK IO를 확인할 수 있습니다.

App 모니터링

각 앱에 대한 CPU, MEMORY, DISK 사용량을 확인할 수 있습니다.

  • 모니터링 내역으로는 CPU, MEMORY, Disk 사용량으로 구성되어있습니다.
알람 확인하기

로그인한 사용자를 기준으로 알람을 생성하여 작업에 대한 내역을 화면에 표시합니다.(임계치 생성기준 WARNING : 80 ~ CRITICAL : 90)

  • 이벤트내역에서 해당 작업에 내용만 사용자에게 보여집니다.
  • 상태는 WARNING 상태와 CRITICAL 상태만을 표시합니다.
이벤트 확인하기

프로젝트 하위 작업에 대한 VM에 대한 이벤트(시작, 중지, 임계치 초과 등)를 확인 할 수 있습니다.

시스템 로그

프로젝트 하위 작업에 대한 시스템 로그 정보를 확인 할 수 있습니다.

  • 로그는 현재 날짜를 기준으로 시스템 로그에 대한 데이터를 확인할 수 있습니다.
  • 우측 상단의 달력아이콘을 클릭하여 사용자가 원하는 날짜의 시스템 로그를 확인 할 수 있습니다.
