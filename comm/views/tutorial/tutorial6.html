저장소 관리

소스코드 저장소를 생성하고 관리합니다.

저장소 관리 계정 생성
  • [저장소 계정 생성] 버튼을 클릭하면 저장소 계정 생성 레이어가 나타납니다.
  • password를 입력 합니다.
  • 비밀번호는 영문,숫자,특수문자(!@$%^&* 만 허용)를 사용하여 8~16자까지, 영문은 대소문자를 구분합니다.
저장소 생성
  • [+] 버튼을 클릭하면 저장소 생성 레이어가 나타납니다.
  • 저장소 타입을 선택합니다.
  • 저장소 이름을 입력합니다. 저장소 이름은 영문 소문자와 숫자 그리고 최소 3자리 이상으로 입력 합니다.
빌드잡 생성
  • 빌드잡 생성버튼을 클릭하면 입력 레이어가 나타납니다.
  • 빌드잡 이름을 입력 합니다.
  • maven의 Goal을 입력합니다. 디폴트로 clean package가 입력 됩니다.
  • pom.xml의 위치를 입력합니다. 디폴트로 trunk/pom.xml이 입력 됩니다.
  • 배포방법을 선택 합니다. PAAS를 선택합니다.
  • PAAS에 배포할 때 배포할 PAAS의 주소를 입력합니다.
  • 배포할 PAAS의 계정을 입력합니다. id/password 형식으로 입력합니다.
  • Org를 입력합니다.
  • Space를 입력합니다.
  • 어플리케이션의 이름을 입력합니다. 디폴트로 빌드잡이름이 입력됩니다.
  • 어플리케이션이 사용할 메모리를 입력합니다. 디폴트로 512가 입력됩니다.
  • 어플리케이션의 hostname을 입력합니다. 디폴트로 빌드잡 이름이 입력됩니다.
  • 어플리케이션이 start하는 시간을 입력합니다. 디폴트로 60(s)가 입력됩니다.
  • PAAS에 push할 어플리케이션 파일의 위치를 입력합니다. 디폴트로 빌드가 끝난 파일은 trunk/target폴더에 위치합니다.
    파일명은 pom.xml에 입력한 artifactId-version.packaging입니다.
    예를 들어 pom파일에 아래와 같이 되어있다면
    <groupId>pipeline</groupId>
    <artifactId>tes</artifactId>
    <version>0.0.1-SNAPSHOT</version>
    <packaging>war</packaging>
    빌드후 생성된 파일은 test-0.0.1-SNAPSHOT.war가 됩니다. 따라서 Application Path에는 trunk/target/test-0.0.1-SNAPSHOT.war을 입력합니다.

빌드 관리

생성한 빌드잡을 빌드하고 상세 내역을 조회 합니다.

빌드잡 리스트
  • 빌드잡의 빌드 정보를 나타냅니다.
빌드
  • 빌드를 클릭하여 빌드를 실행 합니다.
  • 빌드내역을 클릭하여 빌드로그를 확인 할수 있습니다.
  • 이름을 클릭하면 빌드 히스토리페이지로 이동합니다.
빌드 히스토리
  • 해당 빌드잡의 빌드 이력을 나타 냅니다.
  • 변경내역이 존재하는 경우 빌드 변경 내역을 클릭하면 변경내역 페이지로 이동합니다.
빌드 변경 내역
  • 해당 빌드시 변경된 파일목록을 나타 냅니다.
