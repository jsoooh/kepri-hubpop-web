API Gateway 활용하기

API Life Cycle

API Gateway는 손쉽게 API를 생성, 등록, 배포하고 유지 관리, 모니터링 및 보호할 수 있는 서비스입니다.

모든 클라이언트 요청에 대한 end point를 통합하여 API 호출을 수락 및 처리하는 데 관련된 모든 작업을 처리합니다.

  • API 매니저가 서비스 API를 디자인/설계
  • 개발자가 설계를 반영하여 API를 개발하고 테스트
  • 개발된 API를 API 게이트웨이에 배포
  • API를 관리, 용량 및 부하 관리
  • API를 검색하고 사용 개발자 포탈
  • API를 분석하고 모니터링

API 서비스 ID 등록

  • [API 기준 정보 관리]메뉴를 클릭합니다.
  • [서비스 ID 관리]버튼을 클릭하여 [+]를 호출합니다.
  • [서비스 ID 등록]에서 정보들을 순서에 맞게 정확하게 입력합니다.
  • - [서비스 ID]입력, [서비스 이름]설정
  • [저장]버튼을 클릭하여 서비스 ID를 생성합니다.

API 명세 등록

  • [API 명세 관리]메뉴를 클릭하여 [+]를 호출합니다.
  • [API 명세 등록]에서 정보들을 순서에 맞게 정확하게 입력합니다.
  • - [서비스 ID]선택, [API 명세 버전]입력
  • [저장]버튼을 클릭하여 API 명세를 생성합니다.
  • 생성한 API 명세의 [작성]버튼을 클릭합니다.
  • [File] 버튼을 클릭하여 API 명세 파일을 읽어옵니다.
  • [Edit] 버튼을 클릭하여 API 명세를 YAML 형식으로 변환 가능합니다.
  • [저장]버튼을 클릭하여 API 명세를 저장합니다.

API 명세 배포

  • [API 명세 관리]메뉴를 클릭합니다.
  • 생성한 API 명세의 [배포]버튼을 클릭합니다.
  • [step01-상세정보]에서 정보들을 순서에 맞게 정확하게 입력합니다.
  • - (필수사항)[카테고리]선택, [API 배포 이름]입력, [API ID/버전]자동입력, [API 배포 설명]입력, [라우팅 방식]선택
  • - (선택사항)[API 브랜드 포탈 주소]입력, [API 로고 이미지]입력
  • [다음]버튼을 클릭하여 API 상세정보를 저장합니다.
  • [step02-자원정보]에서 API 자원 정보를 선택합니다.
  • - 등록된 자원중 배포할 자원을 선택
  • [다음]버튼을 클릭하여 API 자원 정보를 저장합니다.
  • [step03-핕터]에서 API 필터 정보를 선택합니다.
  • - 배포하면서 적용할 필터기능을 선택
  • [다음]버튼을 클릭하여 API 필터 정보를 저장합니다.
  • [step04-보안]에서 API 보안 정보를 선택합니다.
  • - OAuth 2.0을 선택
  • [다음]버튼을 클릭하여 API 보안 정보를 저장합니다.
  • [step05-배포위치]에서 API 배포위치 정보를 선택합니다.
  • - [개발 및 테스트]선택, [운영]선택
  • [다음]버튼을 클릭하여 API 배포위치 정보를 저장합니다.
  • - 해당 배포위치로 배포진행
  • [step06-배포결과]에서 API 배포결과 정보를 확인합니다.
  • - [개발 및 테스트]선택, [운영]선택
  • [배포한 서비스 보기]버튼을 클릭하여 API 배포 상세정보를 확인합니다.

App 등록

  • [App 등록]메뉴를 클릭하여 [+]를 호출합니다.
  • [API 록]에서 정보들을 순서에 맞게 정확하게 입력합니다.
  • - [프로젝트]선택, [App 이름]입력, [App 설명]입력
  • [저장]버튼을 클릭하여 App 정보를 생성합니다.
  • 생성한 App의 [리스트]를 클릭합니다.
  • App 등록 상세조회의 [API 서비스(범위)선택-서비스추가]를 클릭합니다.
  • App에 서비스할 [API 서비스]를 선택합니다.
  • 선택한 API 서비스의 스코프(범위)를 선택합니다.
  • - [Get]선택, [Post]선택, [Put]선택, [Delete]선택
  • [토큰발급]버튼을 클릭하여 해당 App 토큰정보를 발급합니다.
  • 발급받은 토큰정보로 API를 호출합니다.
  • ex) API 시뮬레이터를 활용한 호출
  • - [Method]입력, [URL]입력
  • - [header정보 Content-Type : application/json]입력
  • - [header정보 Authorization : Bearer + 발급받은 토큰정보]입력
  • 호출한 API의 리턴값을 확인합니다.
