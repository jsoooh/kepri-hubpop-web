해당 archetype를 넥서스에서 선택합니다.

  • Catalog에서 해당 넥서스를 선택합니다.
  • Include snapshot archetypes 체크를 합니다.

wapp <dependency>

</dependency>
wam <dependency>
    <groupId>kr.re.kepri</groupId>
    <artifactId>framework-wam</artifactId>
    <version>0.0.1-SNAPSHOT</version>
</dependency>
bapp <dependency>
    <groupId>kr.re.kepri</groupId>
    <artifactId>framework-bapp</artifactId>
    <version>0.0.1-SNAPSHOT</version>
</dependency>

※ 위와 같이 태그 추가 시 nexus서버에서 PC로 FrameWork jar 파일을 자동으로 다운로드 받습니다.

에러 메시지 관리

프로젝트에서 발생하는 에러를 쉽게 파악하기 위해 Manager Framework에서 에러 메시지 관리 기능을 제공합니다. 에러 메시지 관리 기능을 추가하기 위해 아래와 같이 공통테이블을 생성하고 Logging( 로그백 : logback) 기능을 설정해 주십시오. 개발 플랫폼은 SLF4J(Logback)을 기본 Logging 기능으로 제공합니다.

테이블 명 역 할
LOGGING_EVENT 일반적인 로그 이벤트 정보를 저장하는 테이블
LOGGING_EVENT_PROPERTY 로깅 이벤트가 발생했을 때 context에 저장된 프로퍼티정보와 MDC에 저장된 정보를 저장하는 용도로 사용되는 테이블
LOGGING_EVENT_EXCEPTION 파라미터로 넘겨주는 Exception 객체를 이용하여 stackTrace 정보를 저장하는 용도로 사용되는 테이블
(에러 level이 "ERROR"이상일 경우 적재)


SkeletonWebManagerApp 내에 src/main/resources 하위에 존재하는 logback.xml 파일을 아래와 같이 설정하십시오.

  • logback.xml 파일의 DB Appender 설정을 통해 "LOGGING_EVENT_EXCEPTION" 테이블에 에러 Logging 정보가 적재되도록 설정합니다.

배치 등록

  • 배치에 등록 할 배치 Job을 등록을 한다. (배치 실행 전의 단계)
  • * 배치명은 Job Name과 Job을 수행 할 파일명이 동일하여야 됩니다.

 

스케쥴 등록 관리

  • 배치 등록에서 등록된 Job을 스케쥴 등록 관리에서 일정과 시간을 설정하여 Job이 실행 되도록 관리 설정 합니다.

 

스케쥴 등록

  • 수행할 Job을 특정 시간에 수행하도록 설정합니다. (등록과 동시 설정 시간에 배치 Job이 수행 됩니다.)
