데이터 수집 아키텍처

빅데이터 플랫폼 연계 프로세스
  • DAY + 1 시점에 한전의 레거시시스템 데이터들이 빅데이터플랫폼으로 적재됩니다.
  • DAY + 2 에 연계 요청된 데이터들이 HUB-POP에 전송됩니다.
    이를 데이터 수집 툴인 NiFi가 감지하여 데이터공유영역인 ORACLE과 HDFS에 적재되고, 연구과제는 저장된 데이터를 조회하실 수 있습니다.

현재, DAS, SCADA 등 여러 시스템들이 빅데이터 플랫폼에 적재되고 있습니다.

수집데이터 제공 방안

DB Client를 이용하여 DB 접속 및 조회

적재된 전력마스터 데이터, GIS 설비 데이터들을 제공하는 DB 계정을 통해 접근 가능합니다. 신규 과제는 데이터 카탈로그 포털에 접근하시면 별도의 신청 없이 ORACLE, GIS DB 계정을 생성해드리며, 카탈로그 포털에서 각 DB 정보 및 계정 정보를 조회하실 수 있습니다. 이를 이용하여 DB 툴을 통한 데이터 조회 또는 KDN 프레임워크를 활용한 프로그램 개발에 활용하실 수 있습니다.

ORACLE DB Client를 통한 조회

  • 별도의 신청 절차 없이 ORACLE 계정 생성 제공 (데이터 카탈로그 포털 내 DB / 계정 정보 조회 가능)
  • DB Tool을 이용한 데이터 조회 및 JDBC 연계 개발 가능

GIS DB Client를 통한 조회

  • 별도의 신청 절차 없이 GIS DB 계정 생성 제공 (데이터 카탈로그 포털 내 DB / 계정 정보 조회 가능)
  • DB Tool을 이용한 데이터 조회 및 JDBC 연계 개발 가능
HDFS 브라우징 및 활용 가능

빅데이터 플랫폼을 경유하여 수집하는 데이터들은 연구원 하둡 시스템에도 함께 적재하고 있습니다. 하둡 포털인 ivy를 통해 수집한 데이터를 조회하거나 다운로드 가능합니다.

  • 하둡 포털(ivy)을 통한 수집 데이터 상세 조회
  • 하둡 포털(ivy)을 통한 수집 데이터 조회 및 다운로드 가능(빅데이터 플랫폼을 경유하여 수집된 전체 데이터 조회 및 다운로드 가능)
Table Data API 제공

데이터 카탈로그 포털에서 카탈로그 별 단순 데이터 조회 기능을 제공하고 있습니다. 또한, 카탈로그 별 스키마 정보를 이용한 검색을 통해 데이터를 조회하고 이를 다운로드 받는 기능을 제공합니다.

  • 데이터 카탈로그 포털 내 데이터 카탈로그 별 단순 데이터 조회
  • 데이터 카탈로그 포털 내 데이터 조건 조회
  • REST API 기반의 데이터 조회 및 개발에 활용

사용자 구성

HUB-PoP 플랫폼의 사용자에게 데이터 카탈로그 포털 서비스 제공

데이터 카탈로그 포털의 사용자는 크게 프로젝트 책임자, 프로젝트 개발자, 그리고 시스템 운영자로 나뉘어집니다. 프로젝트 책임자와 개발자는 역할 구분 없이 데이터 카탈로그 포털에 접근 가능하며, 데이터 수집 요청이 가능합니다.
데이터 카탈로그 포털은 별도 로그인 페이지는 제공하지 않습니다. 허브팝 포털에 로그인 하시고, 화면 하단의 데이터 연계 쪽 메뉴를 클릭하시어 접근하실 수 있습니다.

데이터 카탈로그 조회

데이터 카탈로그 조회 (자료 검색)

배전, 송변전 등 업무에 따라 시스템을 분류하고 시스템 별 카탈로그 목록을 트리 형태로 조회할 수 있습니다. 또한, 카탈로그 트리에서 “S” 표시를 통해 해당 연구과제 별 조회 가능한 테이블을 식별하실 수 있습니다.
그리고 오른쪽에는 업무 대분류, 중분류 그리고 테이블 명 등 검색조건을 통해 찾고자 하는 카탈로그를 검색하실 수 있습니다. 이러한 검색을 통해 하단에는 카탈로그 목록이 조회되며, 카탈로그 목록을 통해 업무 대분류, 중분류, 카탈로그 명, 자료 건수 등의 정보를 확인 하실 수 있습니다.

1. 데이터 카탈로그는 2가지 방법으로 조회할 수 있습니다.

1 - 1) 카탈로그 트리 조회

  • 배전, 송변전 등 업무에 따른 시스템을 분류하여 해당 시스템에 속해 있는 테이블을 한 눈에 볼 수 있고, 조회할 수 있습니다.
  • 또한, 카탈로그 트리에서 보여지는 테이블에 대하여 Select 권한을 의미하는 [S] 표시를 통해 현재 로그인한 사용자에 대한 데이터 조회 권한 여부를 확인할 수 있습니다.

1 - 2) 카탈로그 조건 조회

  • 업무 대분류, 중분류, 테이블명 등 검색 조건을 통해 사용자가 찾고자 하는 테이블을 검색할 수 있습니다.

2. 데이터 카탈로그 조회를 통한 목록을 확인 할 수 있습니다.

2 - 1) 카탈로그 목록을 통해 확일 할 수 있는 항목은 다음과 같습니다.

  • 업무 대분류 : 배전, 송변전 등 업무분류
  • 업무 중분류 : DAS, EVDB 등 시스템명
  • 적재 주기: 일,월 등 적재 주기
  • 카탈로그ID: 카탈로그 등록 시 생성되는 ID
  • 데이터 카탈로그명 : 테이블 한글명과 영문명으로 이뤄진 카탈로그 명
  • 자료 건수 : 테이블에 적재되어 있는 건수
  • 최종 적재일 : 테이블에 데이터가 최종 적재된 일시
  • 수집 이력 : 테이블 내 데이터 적재 이력
데이터 카탈로그 조회 (조회 가능 자료 목록)

조회 가능 자료 목록을 클릭하시면 연구과제 별 조회 권한을 부여 받은 테이블을 트리 형태로 제공합니다. 엑셀 다운로드를 통해 조회 가능한 테이블 목록을 엑셀로 다운로드 받을 수 있습니다.

조회 가능 자료 목록은 2가지 방법으로 확인 할 수 있습니다.

1 - 1) 조회 가능 자료 트리 조회

  • 조회 가능 자료 목록 버튼을 클릭하여 팝업 내 트리 형태로 보여지는 Select 권한이 있는 테이블을 볼 수 있습니다.

1 - 2) 전체 자료 다운로드

  • 조회 가능 자료 목록 버튼을 클릭하여 생성된 팝업을 통해 전체 자료 다운로드(EXCEL) 버튼을 클릭하여 Select 권한이 있는 테이블 즉, 조회 가능한 테이블 목록 파일을 다운로드 받을 수 있습니다.
데이터 카탈로그 조회 (조회 가능 자료 목록 - Oracle 계정정보)

해당 화면에서는 과제별로 제공하는 DB 정보를 확인하실 수 있습니다. 허브팝 포털에서 신규 연구과제를 생성 후에 데이터 카탈로그 포털에 최초 접속시 Oracle 계정을 생성합니다. 위 화면에서 계정을 포함한 Oracle DB 접속 정보를 확인하실 수 있습니다.

연구과제별 제공하는 DB 계정정보를 조회하실 수 있습니다

1 - 1) Oracle 계정정보

  • 신규 프로젝트 과제가 데이터 카탈로그 포털에 최초접속 시 Oracle 계정을 생성합니다.
  • 생성된 계정을 통해 요청 데이터를 조회 가능합니다.
데이터 카탈로그 조회 (조회 가능 자료 목록 - GIS 계정정보)

또한, GIS DB 접속 정보도 확인하실 수 있습니다. 마찬가지로 카탈로그 포털에 최초 접속 시 GIS DB 계정을 생성하며 위 화면에서 계정을 포함한 GIS DB 접속 정보를 확인하실 수 있습니다. 위 정보를 확인하시고, pgadmin을 이용하여 설비 DB를 확인 가능하시며 개발에도 활용하실 수 있을거라 보여집니다.

연구과제별 제공하는 DB 계정정보를 조회하실 수 있습니다

1 - 1) GIS 계정정보

  • 신규 프로젝트 과제가 데이터 카탈로그 포털에 최초 접속 시 GIS 계정을 생성합니다.
  • 생성된 계정을 통해 GIS 설비 데이터를 조회 가능합니다.
데이터 카탈로그 조회 (조회 가능 자료 목록 - API 샘플)

조회 가능 자료 목록의 API 샘플을 선택하여 TABLE DATA API 메뉴를 확인할 수 있습니다.
TABLE DATA API를 활용하여 허브팝 내 저장된 자료에 대해 조회, 저장 가능하며, 사용방법에 대한 가이드를 제공합니다.

TABLE DATA API 샘플 메뉴를 통해 기본적인 조회, 저장 기능을 테스트하고 사용 방법을 익힐 수 있습니다.

1 - 1) 샘플 조회

  • 요청 URL 정보로 자료를 조회합니다.
  • 사용방법에는 요청 URL을 작성하는 방법과 예제 소스코드가 제공됩니다.

1 - 2) 샘플 저장

  • 요청 URL 정보로 조회된 자료를 저장합니다.
  • 사용방법에는 요청 URL을 작성하는 방법과 예제 소스코드가 제공됩니다.

2 - 1) 기상정보 조회, 삭제

  • 요청 URL정보 기준으로 기상정보 데이터를 조회합니다.
  • 요청 URL정보로 조회 결과(저장 데이터)에 해당하는 자료를 삭제합니다.

2 - 1) 기상정보 저장

  • 요청 URL정보와 조회 결과(저장 데이터)를 바탕으로 데이터를 저장합니다.
데이터 카탈로그 상세조회
  • 데이터 카탈로그 목록에서 카탈로그 명 클릭시 해당 카탈로그의 상세 정보를 확인하실 수 있습니다.
  • 칼럼명, 커멘트, 사이즈 등의 정보를 확인하실 수 있습니다.
  • 해당 카탈로그 스키마는 빅데이터 플랫폼에서 수집된 자료를 일차적으로 활용하였으나, 일부 누락된 카탈로그는 레거시 담당자를 통해 스키마 정보를 수집하여 카탈로그를 구축하였습니다.

데이터 카탈로그 조회 화면 내 카탈로그 목록에서 ‘데이터 카탈로그명’을 클릭하여 상세 조회를 할 수 있습니다.

데이터 카탈로그 상세조회 (샘플 데이터 조회)

카탈로그 상세 화면에서 데이터 조회 클릭시, 해당 카탈로그 별 적재된 일부 데이터를 조회하실 수 있습니다.

데이터 카탈로그 상세 화면에서 데이터 조회 클릭시 해당 카탈로그의 적재된 데이터를 조회 하실 수 있습니다.

데이터 카탈로그 상세조회 (상세 데이터 조회)

연구과제별 조회 권한이 부여된 카탈로그에 대해 검색을 통해 데이터를 조회하실 수 있습니다.
필수항목은 해당 카탈로그의 키로 식별된 항목이 검색에 제공되며, 키가 아닌 칼럼에 대해서는 선택항목을 추가하시고 검색을 하실 수 있습니다.
이를 통해 하단의 조회 결과로 데이터를 조회하실 수 있으며, 조회결과에 대해 다운로드 기능을 제공할 예정입니다.

데이터 카탈로그 상세 화면에서 데이터 검색 클릭시 해당 카탈로그의 적재된 데이터를 조회 하실 수 있습니다.

1) 검색조건

  • 카탈로그의 스키마 정보를 바탕으로 검색조건을 추가하실 수 있습니다.
  • 필수조건 : 해당 스키마 키
  • 선택항목 : 해당 스키마의 칼럼

2) 조회

  • 검색조건을 바탕으로 데이터를 조회 가능합니다.

데이터 카탈로그 등록

데이터 카탈로그 등록 (직접입력)

카탈로그는 기본 정보와 상세 정보로 나뉘어 등록됩니다.

1) 기본 정보는 아래와 같은 항목으로 이루어져있습니다.

  • 데이터 요청서ID : 데이터 요청서를 통해 요청 받은 테이블에 대한 카탈로그를 등록하는 경우에 필요한 ID
  • 업무 분류 : 업무에 따른 시스템 분류
  • 수집 시스템 : 데이터를 수집해오는 시스템
  • 자료 유형 : 데이터 유형
  • 자료 영문명 : 테이블 영문명
  • 자료 한글명 : 테이블 한글명
  • 적재 주기 : 수집 시스템으로부터 적재되는 주기
  • 카탈로그 제목 : 테이블에 대한 영문명과 한글명의 조합으로, 자료 영문명과 한글명 입력창에 의해 자동 입력
  • 상세 설명 : 테이블에 대한 추가 설명

2) 상세 정보는 아래와 같은 항목으로 이루어져있습니다.

  • 항목명 : 컬럼 한글명
  • 영문명 : 컬럼 영문명
  • 유형 : 데이터 타입
  • 크기(전체) : 데이터 크기
  • 크기(소수부) : Number 타입에서 소수부
  • KEY : 해당 컬럼의 KEY 여부
  • NULL : 해당 컬럼의 NULL 값 여부
  • 항목 설명 : 컬럼 설명

데이터 사용요청 조회

데이터 사용요청 조회화면입니다. 상단은 진행 상태 등 검색 조건으로 구성되며 검색을 통해 요청서 목록을 확인하실 수 있습니다.
아래의 요청서 목록은 요청서ID, 요청 제목, 진행상태로 구성되어 있습니다. 진행상태는 요청, 접수, 작업중, 완료, 반려 이렇게 5단계로 구성됩니다.
요청은 사용자가 데이터 요청서 작성을 완료한 상태를 말합니다.
접수는 운영자는 사용자가 작성한 데이터 요청건을 확인하고 승인한 상태를 말하며, 데이터 카탈로그를 등록하게 됩니다.
작업중은 해당 요청건에 대한 빅데이터 플랫폼과 연계하여 적재가 진행중인 상태를 말하며, 완료는 요청 데이터에 대한 적재, 그리고 권한 부여 작업이 완료된 상태를 말합니다.
완료의 상태인 경우에는 연구과제는 해당 요청건에 대한 데이터를 접근하실 수 있습니다.
반려는 요청서 내용에 잘못 기재된 내역이 있을 경우 반려가 가능합니다.

1. 데이터 사용요청 조회는 전력 SW 연구개발 플랫폼(허브팝)에 저장되어 있는 데이터를 사용 신청 하는 메뉴로서, 아래와 같은 조건을 설정하여 조회 할 수 있습니다.

1 - 1) 요청 제목 : 데이터 요청서 제목

1 - 2) 진행 상태 : 요청, 접수, 작업중, 완료, 반려 총 5단계

  • 데이터 요청서ID : 데이터 요청서를 통해 요청 받은 테이블에 대한 카탈로그를 등록하는 경우에 필요한 ID
  • 요청 : 사용자가 데이터 요청을 마친 상태
  • 접수 : 사용자가 올린 데이터 요청을 관리자가 확인하여 승인하였고, 관리자가 데이터 요청서에 대한 카탈로그 등록을 할 수 있는 상태
  • 작업중 : 카탈로그가 등록된 데이터 요청서에 대해 요청 데이터 적재 또는 요청 데이터 권한 부여 작업이 이루어지는 상태
  • 완료 : 요청 데이터 적재 또는 요청 데이터 권한 부여 작업이 완료되어 관리자가 해당 데이터에 대해 조회 가능을 최종 승인한 상태
  • 반려 : 데이터 요청서에 잘못된 내용, 오류, 요청할 수 없는 데이터에 대한 요청 등이 기재되었을 경우 관리자가 해당 데이터 요청서를 반려한 상태

2. 조회한 데이터 요청서에 대해 요청 유형, 요청서ID, 요청제목, 진행상태, 요청자, 요청일자 항목을 볼 수 있습니다.
