오브젝트 스토리지란?

오브젝트 스토리지는 파일 계층 구조로 데이터를 관리하는 파일 시스템과 달리 데이터를 객체로 관리 하는 데이터 저장 방법으로, 논리적인 스토리지로 볼 수 있기 때문에 물리적 제약 없이 원하는 만큼 공간을 확장시킬 수 있습니다.

주요 기능

  • - 데이터는 파일이나 블록 대신 객체로 노출되고 관리됩니다.
  • - 고유한 주소를 기준으로 개별 데이터 오브젝트를 식별합니다. 더 나은 인덱싱 또는 관리를 위해 메타데이터를 포함할 수 있습니다.
  • - 응용 프로그램이 데이터를 조작 할 수 있도록 프로그래밍 인터페이스를 제공합니다. API 구현은 REST 기반이며 표준 HTTP 호출을 사용할 수 있습니다.

S3 브라우저를 이용한 스토리지 관리

프로젝트 생성 시에 프로젝트별로 접속 정보가 생성됩니다.

1) 사전 준비사항

S3 브라우저 다운로드다운로드

2) 접속 정보 확인

오브젝트 스토리지의 [접속 정보] 에서 접속 정보를 확인합니다.

3) S3 브라우저에 접속정보 입력

① ‘Add new account’ 를 선택합니다.

② 정보 입력 후 ‘Add new account’ 버튼을 클릭하여 생성합니다.

4) bucket(저장소) 내 파일 관리

① 버켓(저장소) 내부 클릭 > 우측 마우스 클릭으로 업로드, 다운로드, 삭제, 이름변경 등 파일 관리가 가능합니다.

② 기본적으로 프로젝트 내에서만 접근 및 사용 가능하나 [Permissions] 탭에서 하단 [Make public] 클릭 시 모든 사용자가 접근 및 사용 가능하도록 권한을 부여할 수 있습니다.

5) 접근 권한이 부여된 파일에 대해서 웹으로 다운로드 가능

[REST Endpoint]/[버켓명]/[파일명]

예)
http://101.55.126.196:7480/aaaaaa/os_cli_pkg.tar.gz
